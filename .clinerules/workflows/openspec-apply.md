# OpenSpec: 適用 (Apply)

承認された OpenSpec の変更を実装し、タスクを同期させます。

<!-- OPENSPEC:START -->

**ガードレール**

- まずはシンプルで最小限の実装を優先し、要求された場合や明らかに不可欠な場合にのみ複雑さを追加してください。
- 変更は、要求された成果の範囲内に厳密に留めてください。
- 追加の OpenSpec 規約や明確化が必要な場合は、`.openspec/` ディレクトリ内の `.openspec/AGENTS.md`（見当たらない場合は `ls openspec` または `openspec update` を実行）を参照してください。

**ステップ**
以下のステップを TODO として追跡し、一つずつ完了させてください。

1. `changes/<id>/proposal.md`、`design.md`（存在する場合）、および `tasks.md` を読み、スコープと受入基準を確認します。
2. タスクを順番に進め、編集を最小限に抑え、要求された変更に集中します。
3. ステータスを更新する前に完了を確認し、`tasks.md` のすべての項目が完了していることを確認してください。
4. すべて作業が完了したら、各タスクが `- [x]` となり現状を反映するようにチェックリストを更新します。
5. 追加のコンテキストが必要な場合は、`openspec list` または `openspec show <item>` を参照してください。

**リファレンス**

- 実装中に提案からの追加コンテキストが必要な場合は、`openspec show <id> --json --deltas-only` を使用してください。
