## ADDED Requirements

### Requirement: NFT のフィルタリング機能

システムは、ユーザーが指定した条件で NFT をフィルタリングする機能を提供しなければならない(SHALL)。

#### Scenario: ジャンルによるフィルタリング

- **GIVEN** ユーザーが検索画面を開いている
- **WHEN** ユーザーが「音楽」ジャンルを選択して検索を実行する
- **THEN** システムは「音楽」ジャンルに属する NFT のみを検索結果として表示する
- **TEST_PLAN**:
  - **Method**: E2E Test
  - **Focus**: ジャンルフィルター選択時の表示結果のフィルタリング
  - **Assertion**: 表示されるすべての NFT カードが「音楽」ジャンルタグを持っていること

#### Scenario: 価格帯によるフィルタリング

- **GIVEN** ユーザーが検索画面を開いている
- **WHEN** ユーザーが価格帯を「1000 円から 5000 円」に設定して検索を実行する
- **THEN** システムは指定された価格帯の NFT のみを検索結果として表示する
- **TEST_PLAN**:
  - **Method**: Unit Test (Search Logic)
  - **Focus**: 価格フィルタリング関数の境界値テスト
  - **Assertion**: 1000 円未満および 5000 円を超える価格の NFT が結果に含まれないこと

### Requirement: NFT のソート機能

システムは、ユーザーが指定した順序で NFT を並び替える機能を提供しなければならない(SHALL)。

#### Scenario: 新着順によるソート

- **GIVEN** ユーザーが検索画面を開いている
- **WHEN** ユーザーがソート順として「新着順」を選択する
- **THEN** システムは NFT を作成日時の降順（新しいものが先）で表示する

#### Scenario: 価格の安い順によるソート

- **GIVEN** ユーザーが検索画面を開いている
- **WHEN** ユーザーがソート順として「価格の安い順」を選択する
- **THEN** システムは NFT を価格の昇順（安いものが先）で表示する
